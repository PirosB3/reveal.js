Platform Overview

Classifier

3 steps
  • Feature extraction: a pipeline of filters, used to extract features that greatly describe a phrase
  • Dimensionality reduction: removes low information features that decrease performance
  • Classification: Uses a supervised learning algorithm to train a classifier, that can be used to predict sentiment

Feature Extraction

How are we going to find features from a phrase?


my_phrase = "Today was such a rainy and horrible day"
  1. Start with a "Bag of Words" representation
  2. Create a pipeline of feature extractors
  3. Pass the representation down the pipeline

Dimensionality Reduction

  • Remove features that are common across all classes (noise)
  • Increase performance of the classifier
  • Decrease the size of the model, less memory usage and more speed

Dimensionality Reduction

How do we find the most informative features?

  1. Calculate the probability each word appears in different contexts
  2. Score each word with a Chi-square test
  3. Take the top N

Dimensionality Reduction

Calculate Frequency distributions

NLTK provides us with these implementations

  • How often have I seen a word?

    
In [25]: fd = FreqDist()
In [26]: [fd.inc("happy") for _ in range(20)]
In [27]: [fd.inc("sad") for _ in range(5)]

In [30]: fd
Out[30]: < FreqDist with 2 samples and 25 outcomes >

  • How often have I seen a word in a specific class?

    
In [32]: cfd = ConditionalFreqDist()
In [37]: cfd["positive"].inc("weather")
In [38]: cfd["negative"].inc("weather")
In [39]: cfd["negative"].inc("sad")

In [43]: cfd["positive"]
Out[43]: < FreqDist with 1 samples and 1 outcomes >
In [44]: cfd["negative"]
Out[44]: < FreqDist with 2 samples and 2 outcomes >

Dimensionality Reduction

Score each word with a chi-square test

Once we have this FreqDist and ConditionalFreqDist, we can score each word on a per-label basis using a chi-square score

Dimensionality Reduction

Take the top N

The bigger the score is, the more informative the feature is

  1. Sort by score in descending order
  2. Take the top N
  3. ..or use a heap ;)

Classification

Now that we can extract features from text, we can train a classifier. The simplest and most flexible learning algorithm for text classification is Naive Bayes

P(label | features) = P(label) * P(features | label) / P(features)
  • Simple to compute = fast
  • Assumes feature indipendence = easy to update
  • Supports multiclass = scalable

Classification

Core

Inter-Process Communication

Why do we need it?
  • Twitter stream is concurrent, up to 50 tps
  • Classifier is slow, no matter how much optimization is made
  • Classifier is a blocking process, Streamer is event-driven

Inter-Process Communication

Creating a distributed system
  • Distribute applications across multiple processes
  • Apps handle a small scope of the responsibility
  • Apps talk to apps via a standardized protocol

Inter-Process Communication

ZeroMQ
